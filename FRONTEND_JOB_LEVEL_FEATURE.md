### 职级管理 vs 职称管理

| 特性 | 职级管理 | 职称管理 |
|------|---------|---------|
| **用途** | 职业发展阶段 | 具体职位头衔 |

| **关联** | 独立存在 | 可关联到职级 |
| **显示** | 表格视图 | 卡片视图 |

## 💡 使用场景示例

### 场景 1：创建职级体系

1. 进入"职级管理"
2. 点击"新增职级"
3. 填写信息：
   ```
   名称：初级
   编码：junior
   系数：1.1
   描述：入职 1-3 年的员工
   排序：1
   ```

### 场景 2：为职称分配职级

1. 进入"职称管理"
2. 点击"新增职称"或编辑现有职称
3. 填写信息：
   ```
   名称：前端工程师
   关联职级：中级 (从下拉框选择)
   ```

### 场景 3：查看职称的职级信息

在职称列表中，每个职称卡片会显示：
- 职称名称
- 关联的职级标签（带颜色）

```javascript
function getLevelTag(code) {
  if (code === 'junior') return 'success'; // 绿色
  if (code === 'mid') return 'warning';    // 橙色
  if (code === 'senior') return 'danger';  // 红色
  return 'info';                           // 灰色
}
```

### 2. 基础系数可视化

```javascript
function getCoefficientTag(coeff) {
  if (coeff >= 1.1) return 'success';
  if (coeff >= 1.0) return 'warning';
  return 'info';
}
```

## 📊 数据流

### 获取职级列表

```javascript
async function fetchLevels() {
  levels.value = await api.get('/api/job-levels');
}
```

### 创建职级

```javascript
await api.post('/api/job-levels', {
  name: '初级',
  code: 'junior',
  base_coefficient: 1.1,
  description: '描述',
  sort_order: 1
});
```

### 创建职称并关联职级

```javascript
await api.post('/api/job-titles', {
  name: '前端工程师',
  level_id: 2  // 选择的职级 ID
});
```

### 组件结构

```
App.vue
├── LoginView.vue

    ├── TeamsView.vue

    ├── TeamMembersView.vue

```

### API 调用封装

所有 API 调用通过 `src/api.js` 统一管理：
- GET `/api/job-levels` - 获取职级列表
- POST `/api/job-levels` - 创建职级
- PUT `/api/job-levels/:id` - 更新职级
- DELETE `/api/job-levels/:id` - 删除职级
- GET `/api/job-titles` - 获取职称列表（含职级信息）

## ⚠️ 注意事项

### 1. 数据验证

**创建/更新职级时**:
- ✅ 必填：名称、编码
- ✅ 可选：基础系数（默认 1.0）、描述、排序

**创建/更新职称时**:
- ✅ 必填：名称
- ✅ 可选：职级 ID（可以为 null）

### 2. 错误处理

- 删除有关联的职级时会提示错误
- 网络请求失败时显示友好提示
- 表单验证失败时阻止提交

### 3. 用户体验

- 加载时显示 Loading 动画
- 空数据时显示 Empty 提示
- 操作成功/失败都有明确反馈
