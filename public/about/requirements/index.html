《需求文档》

项目：Vue3 + Node.js 季度评分管理平台

本系统用于企业内部季度绩效评分，目标是建立公平、透明、可追溯、可复算的评分体系。

1. 项目范围

  • 工号登录与角色权限管理
  • 评分关系配置（按人 / 按组）
  • 评分录入与提交流程
  • 多算法组合评分计算
  • 小组评级与结余滚存
  • Excel 一键导出（含可复算公式）

2. 角色与权限

  • 普通员工：按分配任务打分
  • 小组领导：配置评分关系
  • 管理员：全局配置、等级确认、导出、开关控制

3. 全局开关

  • ENABLE_TRIM：是否去除最高最低分
  • ENABLE_CALIBRATION_TYPE：松紧校准类型（TRUE=平均值，FALSE=Z-Score）
  • ENABLE_LEVEL_FIXED_COEFFICIENT：职级调整类型（TRUE=固定系数，FALSE=职级Z）
  • ENABLE_RATIO：是否启用小组比例评级
  • ENABLE_BALANCE：是否启用结余参与基数计算

4. 计算流程

  1. 收集原始分
  2. 可选去极值（打分人数大于等于 4 时去 1 高 1 低）
  3. 松紧校准（二选一）
  4. 职级调整（二选一）
  5. 小组评级与结余滚存

5. 算法组合

  • 平均值松紧 + 固定系数
  • 平均值松紧 + 职级Z
  • Z-Score 松紧 + 固定系数
  • Z-Score 松紧 + 职级Z

6. 导出需求

导出 Excel 必须包含 4 个 Sheet：

  • 系统配置
  • 结余表
  • 原始分明细（长格式）
  • 评分总表（全流程公式）

导出结果要与后台计算一致，切换开关后公式与结果同步变化。

7. 技术与部署

  • 前端：Vue3 + Vite + Element Plus
  • 后端：Node.js + Express
  • 导出：ExcelJS
  • 数据库：SQLite / MySQL
  • 部署：前端 Vercel，后端 Render
